常用命令

git init

在当前目录初始化一个git仓库

git clone

git clone https://xxx.git 或者 git@xx.com/project.git

从远程仓库克隆到本地

git remote add origin https://xxx.git 或者 git@xx.com/project.git

本地仓库关联到远程仓库

git remote set-url origin origin https://xxx.git 或者 git@xx.com/project.git

本地仓库修改关联远程仓库的地址

git pull

从远程仓库拉取最新内容并合并到当前分支中

git add

将文件提交到暂存区里面

git commit

将暂存区里的改动提交到本地仓库，一般通过 -m ‘msg’ 方式来给该提交添加说明备注

git push

将本地仓库推送到远程仓库中

如果是本地仓库关联到远程仓库的第一次push，需要添加 –set-upstream origin master

如：git push --set-upstream origin master

git config –global credential.helper store

解决git pull/push 时每次都要输入用户名和密码的问题，配置完成后执行git pull需要再次输入密码，之后就可以不用输入了

git checkout

git branch

git cherry-pick

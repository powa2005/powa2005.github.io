Nginx 配置 Web SPA应用时刷新404问题

By xiangfengw

2017-09-11

Contents
  1. 1. 问题
  2. 2. 原因
  3. 3. 解决

问题

nginx配置spa应用如vue，react等框架web时候，经常会遇到刷新404的问题

例如：一个基于React的网站页面url是 xx.com/web/info，此时刷新或者跳转到/web/info2就会显示404

原因

一般这种网站都是一个html，然后根据页面路由来渲染不同的组件
所以刷新的时候请求到nginx服务器，服务找不到对应url的资源，自然就会报404错误

解决

nginx配置路径里加一行try_files $uri /web/index.html;

try_files的意思是在找不到请求url对应的资源，就会尝试去需找$uri指定的路径，类似一个内部重定向的操作
